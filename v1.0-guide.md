# Praxiom v1.0 Guide

This guide documents all user-visible and admin/teacher functionality in the Praxiom web application. It is derived from the current codebase and schema in this repo.

## Product Overview
Praxiom is a contest platform for math (or similar) problems with three contest modes (individual, team, relay), a full submission system, ratings, leaderboards, user profiles, groups for schools, a blog with comments and votes, and a curated resources area. Admins and teachers have management tooling; students and general users can register, compete, practice, and participate in community features.

## Roles and Permissions

### Anonymous (not signed in)
- View public pages: home, contests, contest pages (view-only), standings, submissions (answers hidden unless solved), blog posts, resources (public only), leaderboard, user profiles (limited data).
- See "school-only" resources only when authenticated and in the correct school.

### Authenticated user (default)
- Register for individual contests.
- Join/participate in team and relay contests.
- Submit answers, use hints (with XP cost), and upsolve after a contest ends.
- Create blog posts, comment, and vote.
- View own profile data (including full answers to own submissions).
- Access group pages and group chat if part of a group.

### Teacher
- Has all user permissions.
- Is associated with a School and a Group (OrgGroup).
- Approves or rejects students who register under the teacher's group.
- Can view group-specific contest results and student activity.
- Can create and manage resources (including school-only resources).
- Can edit their group bio.

### Admin
- Full access to all features.
- Approves or rejects new users (global pending approvals).
- Creates and manages contests and problems.
- Calculates ratings and closes contests.
- Creates and edits site-wide themes.
- Creates and edits announcements or blog posts (and can delete any post).
- Can see answers in All Submissions.

## Core Features

### 1) Contest System

#### Contest Types
- Individual
- Team
- Relay

Each contest has:
- Title, description, lore, and optional theme.
- Start and end timestamps (live window).
- Visibility, problem list, and settings like sequential locking and rating eligibility.

#### Registration
- Individual contests require explicit registration per user.
- Team and relay contests use team creation and membership rather than individual registration.

#### Live Contest Rules
- Submissions are accepted only while the contest is live.
- Sequential contests enforce a lock: you must solve the previous problem before seeing the next (individual only).
- For relay contests, problems are assigned to specific team members; only the assigned member may submit for their slot.
- Team and relay contests allow teams; team leaders can manage membership and relay order.

#### Upsolve (Post-Contest)
- After the contest ends, users can submit upsolve attempts (not counted for standings).
- Upsolve submissions appear in the "All Submissions" page and are flagged as UPSOLVE.

#### Scoring and Standings
- Standings use an ICPC-style model:
  - Each solved problem contributes to score.
  - Time penalties applied based on submission time and incorrect attempts.
- For team/relay contests, standings are per team.
- For individual contests, standings are per user.
- Standings show per-problem status and solve counts.

#### Ratings
- Ratings are computed for individual contests by admin action.
- Each user's rating history is stored and shown on profiles.
- Rating deltas appear in standings after calculation.

#### Submissions and Answer Visibility
- Contest submissions page shows all in-contest submissions.
- All Submissions page includes contest and upsolve attempts.
- Answers are hidden unless the viewer has solved the problem (or is admin on All Submissions).

### 2) Problem System
- Each contest has a list of problems ordered by ID.
- Problems have title, statement (Markdown + LaTeX), answer key, points, and optional hints.
- Submissions compare answers case-insensitively with trimmed whitespace.
- Problem pages show statement, stats, and submission interface.
- Practice mode via the Problems archive allows submissions without rating impact.

### 3) Hints and XP
- Users earn XP for first-time correct solves.
- Hints can be purchased during live contests.
- Hint cost is half the problem's point value.
- Buying a hint deducts XP and records the hint reveal.

### 4) Team and Relay Management
Team and relay contests provide a complete team workflow:
- Create team (leader is creator).
- Invite users by username.
- Accept/decline invites.
- Users can request to join a team; leaders approve or reject.
- Leaders can remove members or transfer leadership.
- Relay contests include ordered slot assignments (relay order) per member.

### 5) Leaderboards
- Global leaderboard shows user rankings by rating/score.
- Group leaderboard shows rankings of school groups.
- Contest standings provide contest-specific rankings (see above).

### 6) Blog and Announcements
- Blog index includes announcements and user posts.
- Authenticated users can create posts.
- Posts support Markdown and LaTeX.
- Comments are supported per post.
- Voting system (up/down) available on posts.
- Admins can edit or delete any post; authors can edit their own posts.

### 7) Resources Library
- Resources can be public or school-only.
- Teachers and admins can create and manage resources.
- Access to school-only resources is restricted to users in the same school.

### 8) Groups and School System
- Users can belong to a school and group (OrgGroup).
- Teachers have their own group for approvals and student management.
- Group pages show members and group info.
- Group chat supports messages and read receipts.

### 9) User Profiles
- Public profile pages show user rating chart, activity heatmap, and recent submissions.
- Submission answers are hidden to other users unless they solved the same problem.
- Users can edit their own description.

### 10) Themes
- Theme system supports built-in and custom themes.
- Themes include fonts, colors, and CSS variables.
- Admin panel lets admins create and edit themes.
- Theme selection affects navbar/footer colors and overall styling.

## Key Pages and User Flows

### Home
- Announcements and site stats.
- Links to contests and registration.

### Authentication
- Credentials-based sign-in.
- Account must be approved unless admin.

### Contests
- `/contests`: list upcoming, live, and past contests.
- `/contests/[id]`: contest overview, problems list, registration or team actions.
- `/contests/[id]/problems/[problemId]`: problem statement and submission UI.
- `/contests/[id]/standings`: contest rankings.
- `/contests/[id]/submissions`: contest-only submissions (answers hidden unless solved).
- `/contests/[id]/all-submissions`: contest + upsolve submissions (admins can see all answers).

### Problems Archive
- `/problems`: all problems across contests for practice.

### Blog
- `/blog`: posts and announcements.
- `/blog/new`: create a new post (authenticated only).
- `/posts/[id]`: post detail, comments, votes.

### Resources
- `/resources`: resource list.
- `/resources/[id]`: resource detail.
- `/resources/new`: create a resource (teacher/admin).

### Leaderboard
- `/leaderboard`: toggle between user and group rankings.

### Groups
- `/groups/[id]`: group details and members.
- `/groups/[id]/chat`: group chat.

### User
- `/user/[username]`: profile page, rating history, activity, recent submissions.

### Teacher Dashboard
- `/teacher`: pending student approvals, student list, contest results.

### Admin Panel
- `/admin`: approvals and management hub.
- `/admin/contests/new`: create contest.
- `/admin/contests/[id]/edit`: edit contest, manage problems, delete contest.
- `/admin/contests/[id]/problems/[problemId]/edit`: edit problem.
- `/admin/posts/new`: create announcements/posts.
- `/admin/themes`: manage themes.
- `/admin/themes/new`: create theme.
- `/admin/themes/[slug]/edit`: edit theme.

## Data Model Summary (Core Entities)

- User: username, email, hashed password, rating, XP, role flags, school/group links.
- School: name, email domain for validation.
- OrgGroup: school-linked group, teacher owner.
- Contest: type, timing, description, theme, sequential mode, ratings flag.
- Problem: contest-linked, statement, answer, points, hint data.
- Submission: answer, correctness, upsolve flag, timestamps, links to user/problem/contest.
- Registration: user contest registration (individual contests).
- RatingHistory: user rating points over time.
- HintReveal: user-to-problem hint purchases.
- ContestTeam / ContestTeamMember / ContestTeamInvite / ContestTeamJoinRequest: team and relay membership management.
- BlogPost / Comment / Vote: content and engagement.
- Resource: public or school-only learning materials.
- GroupMessage / GroupMessageRead: group chat and read tracking.
- Theme: configurable style presets.

## Submission and Rate Limiting Rules
- Submissions: 30 per minute per user.
- Hint requests: 10 per minute per user.
- Blog comments and votes are rate limited to reduce abuse.
- Registration and school lookup endpoints are also rate limited.

## Security and Privacy Notes
- Users are not approved until an admin (or teacher for students) approves them.
- Answer visibility is restricted: users must solve a problem to see others' answers (admins can see more in All Submissions).
- School-only resources require matching school membership.
- Group chat is limited to group members.

## Admin and Teacher Workflows

### Admin Workflows
- Approve or deny new users.
- Create, update, and delete contests and problems.
- Trigger ratings calculation after contest ends.
- Manage blog posts and announcements.
- Manage themes.

### Teacher Workflows
- Approve or deny students associated with their group.
- View students and contest results.
- Create and manage resources for their school.
- Edit group information.

## Operational Notes
- Uses Next.js App Router with server components and route handlers.
- Prisma + Postgres for persistence.
- NextAuth credentials provider for authentication.
- KaTeX for math rendering in Markdown.
- Tailwind v4 configured, but UI is mostly custom CSS and inline styles.

## Known UX Behaviors and Edge Cases
- Sequential locking applies only to individual contests.
- Relay order controls which team member may submit for a given problem.
- Standings and live contest banners are dynamic and reflect contest time window.
- Answers are hidden in submissions lists unless the viewer has solved the problem.
- Upsolve submissions never affect standings or ratings.

## Appendix: Public and Authenticated API Overview

### Auth
- `/api/auth/[...nextauth]` credentials login.

### Registration and Admin Approval
- `/api/register` create account (school domain validation for teachers).
- `/api/admin/users` approve or deny user.
- `/api/teacher/pending` list pending students.
- `/api/teacher/users` approve or deny student.
- `/api/teacher/students` list students.

### Contests and Problems
- `/api/contests` create contest (admin).
- `/api/contests/[id]` get/update/delete contest.
- `/api/contests/[id]/register` register for contest.
- `/api/contests/[id]/calculate-ratings` compute ratings and end contest (admin).
- `/api/contests/[id]/problems` create problem (admin).
- `/api/contests/[id]/problems/[problemId]` get/update/delete problem (admin).

### Submissions and Hints
- `/api/submissions` submit answer (contest and upsolve).
- `/api/hints` purchase/reveal hint.

### Teams and Relay
- `/api/contests/[id]/teams` list/create teams.
- `/api/contests/[id]/teams/[teamId]` get team details, delete team.
- `/api/contests/[id]/teams/[teamId]/members/[memberId]` remove member.
- `/api/contests/[id]/teams/[teamId]/invites` create invite.
- `/api/contests/[id]/teams/[teamId]/invites/[inviteId]` accept/decline/revoke.
- `/api/contests/[id]/teams/[teamId]/join-requests` request to join.
- `/api/contests/[id]/teams/[teamId]/join-requests/[requestId]` accept/reject.
- `/api/contests/[id]/teams/[teamId]/relay-order` update relay assignments.
- `/api/contests/[id]/teams/[teamId]/transfer-leadership` transfer leadership.
- `/api/contests/[id]/my-invites` list user's pending invites.

### Blog and Community
- `/api/posts` list/create posts.
- `/api/posts/[id]` get/update/delete post.
- `/api/posts/[id]/comments` add comment.
- `/api/posts/[id]/vote` get or toggle vote.

### Resources
- `/api/resources` list/create resources.
- `/api/resources/[id]` get/update/delete resource.

### Groups
- `/api/groups` list groups.
- `/api/groups/[id]` group detail and edit bio.
- `/api/groups/[id]/chat` list/send group chat messages.

### User
- `/api/user/profile` update description.
- `/api/user/xp` get XP.
- `/api/user/unread-messages` unread group chat count.
- `/api/user/live-contests` active contests for live banner.
